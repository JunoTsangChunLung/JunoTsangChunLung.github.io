GuacaHoPE 的前身

上回說到 GuacaHoPE 是 Aphache Guacamole 的應用的延伸。 Apache Guacamole 是一款可遙距操控桌面的應用, 在此之上, 如果還想增加一些額外的 feature, 便要透過一些現有的applcation and framework 去處理, 例如 User, Connections management, Authentication 等等。 將應用變成一款現代的 application, 不論是 web 還是 mobile。

Guacozy

其實 GuacaHoPE 是參考到在 GitHub 上的一個 repo, 叫 Guacozy。
Guacozy 超級厲害, 最起碼在我現在的水平看是如是, 他在 Guacamole 的基礎上增加了 connection management, user management, ticket management 和 folder management。非常強大, 非常美麗。

Guacozy 的組成大致上可以分為四個。

Frontend 前端

使用 React Framework, 都是老生常談了, 以後有機會可以談談, 用 React 的好處便是可以輕易地從伺服器端(server side) 處理好網頁然後一次過compile and export。

Backend 後端

使用 Python Django Framework, 我個人亦十分喜歡 Django 這個 Framwork, 內置了十分強大的 Template 和 Admin Panel, 這亦是為甚麼 Guacozy 可以做到十分多種不同種類的 Management。但是 Folder, Ticket, Session等等的概念仍然需要一定的工程師底子, 這亦是我十分欣賞的地方。

Database 伺服器

用了 Relational Database 和 Django 內置的 Sqlite, 十分厲害的是他用 Relational Database 寫了一個十分嚴謹和複雜的 Schema, 這裏先賣個小關子, 有興趣的朋友可以到他的 database 查看, 看看能不能看出他的整個 Database 的 Schema。

Guacd

這個就是 Apache Guacamole 提供給我們的一個 Agent, 他幫我們處理了十分多遙距連線時要處理到的複雜問題, 讓我們可以用他的 API 去做控制, 只要好好地在前端和後端設定好需要的 setting, 剩下的便是 Guacd 的活了, 設定亦非常簡單, 可以透過 Docker Container 直接在電腦或機械內部創造一個空間給 Guacd Server, 然後透過 Docker Command expose port 讓後端可以連結。

為何還要 GuacaHoPE

有朋友仔可能會問, 既然 Guacozy 都如此強大, 為什麼仍要創建一個GuacaHoPE？

原因是, 我在用 Guacozy 增加一些額外的 feature 的時候有十分多的困難, 除了我的實力問題外, 用 Python 去進行開發的問題亦浮現出來, 我難費了大量時間去閱讀和理解 Guacozy 的內部結構, 最後雖然成功增加到我想要的 Feature 的初型, 但是再之後我還想進行大量的改動, 為了使開發更順利, 我便想到用 Nodejs 和 MongoDB 去進行後端的開發, 一來是我自己親手創建, 正所謂「沒有人比你自己更明白你自己的編碼」於是 GuacaHoPE 便誕生了。

之後我會先講下 Guacozy 的一些 Feature 然後再講 GuacaHoPE 如何根據 Guacozy 去進行改造, 哈哈, 畢竟我也是這樣做的, 順着自己的思維邏輯去寫部落好像比較好。

因為這是我第一次寫部落格, 所以我仍然在探索如何可以做得更好。

我們後會有期！
